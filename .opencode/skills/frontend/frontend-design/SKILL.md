# Frontend Design

建立具有辨識度、生產級品質的前端介面。成果必須能實際運作，並具備清楚的設計觀點、良好的互動細節、響應式版面與完整的視覺品質。

避免通用、模板化、缺乏脈絡的 AI 風格設計，例如制式 SaaS 卡片版面、白底紫色漸層、普通字體、無意義裝飾圖形與只在單一螢幕尺寸看起來正常的版面。

## 核心原則

1. 先理解產品目的、使用者與使用情境。
2. 選擇一個明確且有記憶點的美學方向。
3. 建立真正可用的介面，而不是只做靜態展示。
4. 優先遵循現有專案的框架、元件風格、CSS 方法與設計系統。
5. 加入必要的互動狀態、空狀態、載入狀態與錯誤狀態。
6. 完成前檢查響應式、可讀性、溢出、對比與互動細節。

## 設計思考

在實作前，先明確定義：

- **目的**：這個介面幫助使用者完成什麼事？
- **受眾**：誰會使用？使用頻率高不高？
- **語氣**：選擇一個鮮明方向，例如極簡、工業、編輯雜誌、奢華、粗獷、復古未來、科學儀表、玩具感、沉穩營運工具或高密度專業介面。
- **記憶點**：使用者看一眼後會記住什麼？
- **限制**：尊重框架、效能、無障礙、品牌、既有樣式與可用素材。

克制的設計也必須有觀點。不要退回中性模板。

## 視覺標準

### 字體

- 避免 Arial、Roboto、Inter、系統預設字體等缺乏特色的選擇，除非專案既有設計系統要求。
- 優先選擇有個性的標題字體，並搭配易讀的內文字體。
- 用字重、尺寸、間距與節奏建立層級。
- 所有文字在手機與桌面都必須清楚可讀。
- 不使用容易造成溢出的 viewport 字體縮放。
- 字距必須有意圖；不要隨意使用負字距。

### 色彩

- 使用 CSS 變數管理主色、背景、文字、邊框、陰影與強調色。
- 建立一個明確主氛圍，搭配少量銳利重點色。
- 避免整個畫面只有同一色系的深淺變化。
- 避免常見的白底紫色或藍紫色漸層 AI 風格。
- 確保文字、按鈕、表單、停用狀態都有足夠對比。

### 版面

- 版面應該像被設計過，而不是元件堆疊。
- 可使用非對稱、重疊、留白、高密度資訊、斜向節奏或打破網格的構圖。
- 不要在卡片裡再放卡片。
- 工具型或應用型產品的第一屏應該是實際工作介面，不是行銷 Hero。
- 固定格式元素如工具列、棋盤、圖表、面板、卡片與控制列，需設定穩定尺寸，避免 hover 或內容變化造成跳動。
- 必須同時考慮手機、平板與桌面版面。

### 動效

- 動效要有目的，不要到處加入無關動畫。
- 優先設計一個有記憶點的進場、切換或狀態轉換。
- 靜態 HTML/CSS 優先使用 CSS 動畫。
- 若專案已有動畫函式庫，可沿用既有方案。
- 尊重 `prefers-reduced-motion`。
- 優先動畫化 opacity、transform、filter 或自訂屬性，避免造成 layout thrashing。

### 視覺質感

- 可使用顆粒、細邊框、陰影、遮罩、紋理、圖樣、圖片處理或材質感，但必須服務整體概念。
- 不要加入隨機光球、漂浮 blob、無意義 SVG 波浪或抽象裝飾。
- 需要圖片時，優先使用與產品、場景、物件、資料或主題直接相關的素材。
- 視覺工具、遊戲或沉浸式體驗應該建立真實場景與資產，不依賴空泛占位圖。

## 互動品質

介面必須像完整產品，而不是靜態截圖。

- 按鈕需有 hover、active、focus、disabled 與必要的 loading 狀態。
- 表單需有標籤、驗證、錯誤訊息與可見焦點。
- 儀表板需有合理範例資料、篩選、選取狀態與空狀態。
- 導覽需清楚顯示目前位置。
- Icon-only 按鈕需有可理解的圖示與無障礙標籤。
- 熟悉任務使用熟悉控制：開關用 toggle、模式用 segmented control、數值用 slider/input/stepper、視圖切換用 tabs、選項集合用 menu。

## 可用性與響應式

- 使用語意化 HTML。
- 保留鍵盤操作與清楚 focus 樣式。
- 為 icon-only 控制提供 accessible name。
- 避免文字重疊、裁切或溢出。
- 長字串、數字、標籤與動態內容不可破壞版面。
- 至少檢查窄手機、一般桌面與寬螢幕尺寸。
- 所有重要資訊在手機上仍需可讀且可操作。

## 實作規則

- 優先沿用現有專案的技術棧與風格。
- 不引入不必要的新依賴。
- 使用 CSS 變數維持一致性。
- 只有在能減少真實重複或提升可維護性時才抽象元件。
- 裝飾效果需可維護，不能讓程式碼失控。
- 避免假資料過於空泛，讓介面看起來像模板。
- 不在畫面上寫一堆說明功能的文字，除非產品本身需要 onboarding。

## 反模式

避免：

- 制式 SaaS 卡片牆與模糊指標。
- 白底紫色、藍紫色漸層作為主要設計。
- 工具型產品卻做成大型行銷 Hero。
- 卡片包卡片。
- 隨機光球、blob、抽象波浪與無意義 SVG。
- 看起來像瀏覽器預設的字體。
- 低對比灰字。
- 只在單一尺寸正常的版面。
- 沒有互動狀態的靜態 mockup。
- 裝飾比功能更搶眼。

## 完成前檢查

交付前確認：

- 第一屏清楚呈現實際產品或主要工作流程。
- 美學方向一眼可辨識。
- 字體、色彩、間距、動效一致。
- 手機版不是事後補上的。
- 文字沒有溢出、重疊或不可讀。
- 互動元素有清楚狀態。
- 頁面沒有明顯 console error。
- 結果不像通用 AI 模板。

## 回覆格式

完成後簡短說明：

- 建立或修改了什麼。
- 主要設計方向。
- 如何執行或查看。
- 已檢查哪些項目。
