Epic：個人行事曆管理
1. 建立行程

User Story
作為一位使用者，
我想要新增一筆個人行程，
以便記錄重要活動並安排我的時間。

Acceptance Criteria

使用者可以輸入行程標題。
使用者可以選擇開始時間與結束時間。
使用者可以選擇日期。
使用者可以新增地點與備註。
儲存後，行程會顯示在行事曆中。
2. 編輯行程

User Story
作為一位使用者，
我想要修改已建立的行程，
以便在時間、地點或內容變更時更新資訊。

Acceptance Criteria

使用者可以點選既有行程查看詳細內容。
使用者可以修改標題、時間、日期、地點與備註。
修改後的內容會正確儲存。
行事曆畫面會即時顯示更新後的資料。
3. 刪除行程

User Story
作為一位使用者，
我想要刪除不需要的行程，
以便保持行事曆整潔。

Acceptance Criteria

使用者可以刪除單一行程。
刪除前系統會顯示確認提示。
使用者確認後，行程會從行事曆中移除。
取消刪除時，行程會保留。
4. 查看行事曆

User Story
作為一位使用者，
我想要以日、週、月視圖查看行事曆，
以便快速掌握不同時間範圍內的安排。

Acceptance Criteria

使用者可以切換日視圖、週視圖與月視圖。
不同視圖會顯示對應時間範圍內的行程。
使用者可以切換上一天、下一天、上一週、下一週、上一月、下一月。
當天日期會有明顯標示。
5. 行程提醒

User Story
作為一位使用者，
我想要設定行程提醒，
以便在活動開始前收到通知。

Acceptance Criteria

使用者可以選擇提醒時間，例如 5 分鐘前、15 分鐘前、1 小時前、1 天前。
到達提醒時間時，系統會發送通知。
使用者可以關閉或修改提醒設定。
沒有設定提醒的行程不會發送通知。
6. 重複行程

User Story
作為一位使用者，
我想要建立重複性行程，
以便管理固定發生的活動，例如每週會議或每日運動。

Acceptance Criteria

使用者可以設定行程重複頻率，例如每日、每週、每月。
使用者可以設定重複結束日期。
系統會依照規則自動產生重複行程。
使用者可以選擇修改單次行程或整個系列。
7. 搜尋行程

User Story
作為一位使用者，
我想要搜尋行事曆中的行程，
以便快速找到特定活動。

Acceptance Criteria

使用者可以輸入關鍵字搜尋行程。
搜尋結果會比對行程標題、地點與備註。
搜尋結果會顯示符合條件的行程列表。
使用者可以點選搜尋結果查看詳細內容。
8. 分類行程

User Story
作為一位使用者，
我想要替行程設定分類，
以便區分工作、生活、學習與其他活動。

Acceptance Criteria

使用者可以為行程選擇分類。
不同分類可以用不同顏色顯示。
使用者可以依分類篩選行程。
未分類行程會歸入預設分類。
9. 衝突提示

User Story
作為一位使用者，
我想要在新增或修改行程時收到時間衝突提醒，
以便避免安排重疊活動。

Acceptance Criteria

當新行程與既有行程時間重疊時，系統會顯示提示。
使用者仍可選擇繼續儲存。
系統會清楚顯示衝突的行程名稱與時間。
沒有衝突時可直接儲存。
10. 同步行事曆

User Story
作為一位使用者，
我想要將個人行事曆與其他裝置同步，
以便在手機、平板與電腦上查看相同的行程。

Acceptance Criteria

使用者登入後可以同步行程資料。
新增、修改或刪除行程後，其他裝置會更新。
離線時可查看既有行程。
網路恢復後，系統會同步離線期間的變更。
